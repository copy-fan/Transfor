Workflow

从文件到飞书 Base 的固定路径

skill 把容易出错的步骤固定成脚本：先预检，再创建 Base，最后导入记录、视图和看板。 如果任一步失败，会留下 JSON 日志，方便继续而不是重来。

  1. Preview识别主表、辅助表、字段类型和可生成组件。
  2. Create用 lark-cli 创建飞书 Base 和第一张主表。
  3. Import按 200 行分批写入记录，其他 sheet 作为辅助表导入。
  4. Analyze自动创建高潜、低效、标签补齐视图和消耗分布图。

Features

面向素材复盘，也适合通用表格迁移

字段类型推断

自动识别文本、数值、金额、百分比、日期时间、单选和多选字段。

多 sheet 导入

主数据进入核心表，字段字典、原始数据、视图建议等保留为辅助表。

运营视图

根据 ROI、消耗、脚本风格、开头钩子和标签字段生成行动队列。

看板组件

自动生成消耗占比饼图，帮助快速判断哪类脚本和钩子最吃预算。

Usage

两条命令完成预检和导入

先用 dry preview 看计划，再执行真实导入。需要飞书 CLI 已安装并完成授权。

python3 scripts/feishu_base_from_excel.py \
  --excel /path/to/workbook.xlsx \
  --prepare-only

python3 scripts/feishu_base_from_excel.py \
  --excel /path/to/workbook.xlsx \
  --base-name "千川素材跑量分析"
